# Medieval Runner

Динамичный бесконечный раннер в средневековом сеттинге, разработанный на Unity с использованием событийно-ориентированной архитектуры и новой системы ввода (New Input System).

## 1. Описание сущностей и игровых функций

### Игрок (Player)
*   **Перемещение:** Свободное перемещение по плоскости (не по фиксированным линиям) с использованием `CharacterController`.
*   **Прыжок:** Реализован одинарный прыжок с проверкой на приземление.
*   **Здоровье:** Система здоровья с настраиваемым максимальным значением. При получении урона активируется режим неуязвимости (i-frames) на заданное время.
*   **Урон:** Игрок получает урон при физическом столкновении с препятствиями (`OnCollisionEnter`).
*   **Падение:** Если позиция игрока по оси Y опускается ниже −2, он мгновенно погибает (теряет всё здоровье).

### Механика ускорения (Boost)
*   **Активация:** Удержание клавиши `SHIFT` (или кнопки геймпада).
*   **Эффект:** 
    *   Скорость движения мира увеличивается в **2 раза**.
    *   Начисление очков (Score) увеличивается в **2 раза**.
*   **Сброс:** При отпускании клавиши скорость и множитель очков возвращаются к базовым значениям.

### Препятствия (Obstacles)
В игре реализовано два типа препятствий, использующих физику для взаимодействия:
1.  **Rolling Boulder (Валун):** Спавнится на высоте, падает под действием гравитации и катится в сторону игрока. Требует уклонения в стороны.
2.  **Rolling Log (Бревно):** Катится по всей ширине пути. Требует своевременного прыжка.
*   **Логика:** Препятствия имеют ограниченное время жизни (Lifetime) и автоматически уничтожаются. Настройка параметров (скорость, урон, высота спавна) производится через `ScriptableObject`.

### Мир и пол (World/Floor)
*   **Бесконечный путь:** Реализован через систему сегментов (`RunSegment`). 
*   **Генератор:** `RunGenerator` перемещает сегменты навстречу игроку. Когда сегмент выходит за пределы видимости (Z-граница), он переносится в конец очереди, создавая эффект бесконечного движения.

## 2. Архитектурный подход

Проект построен на **Event-driven (событийно-ориентированной) архитектуре**. Это минимизирует жесткие связи между модулями (Loose Coupling) и делает систему легко расширяемой.

### Основные принципы:
*   **Отсутствие статических хабов:** Используются стандартные C# события (`event Action`).
*   **Инъекция зависимостей:** Ссылки на необходимые модули передаются через Инспектор.
*   **Изоляция:** Каждый модуль отвечает только за свою узкую задачу.

### Структура модулей:
*   **Core:** Управление глобальным состоянием игры (`GameState`: Menu, Playing, GameOver) через `GameManager`.
*   **Input:** Обертка над Unity New Input System. Переводит низкоуровневые нажатия в игровые события.
*   **Player:** Логика движения, прыжков и обработки здоровья.
*   **World:** Управление скоростью мира и рециркуляцией сегментов пола.
*   **Obstacles:** Система спавна и поведения препятствий на основе данных из `ScriptableObject`.
*   **Bonuses:** Бонусы подбираются через триггер. `HealthBonus` восстанавливает здоровье, `InvulnerabilityBonus` даёт временную неуязвимость. Настройка через `ScriptableObject`.
*   **Score:** Расчет очков в реальном времени с учетом множителей.
*   **Save:** Сохранение рекорда через `PlayerPrefs` между сессиями.
*   **UI:** Модули отображения (Score/Health/меню), подписанные на события соответствующих менеджеров. При Game Over время замораживается (`timeScale = 0`), но UI-кнопки продолжают работать.

## 3. Взаимодействие модулей

Взаимодействие между модулями происходит через подписки на события:

1.  **Boost Flow:**
    `PlayerInputHandler` (событие) → `WorldMover` (меняет скорость) & `ScoreManager` (меняет множитель).
2.  **Movement Flow:**
    `PlayerInputHandler` → `PlayerMovement` (задает вектор движения).
3.  **Damage Flow:**
    `Obstacle` (коллизия) → вызывает `TakeDamage` у `PlayerHealth`.
4.  **UI Flow:**
    `ScoreManager` & `PlayerHealth` (события изменения) → `UIViews` (обновляют текст на экране).
5.  **Fall Flow:**
    `PlayerFallDetector` (Y < −2) → `PlayerHealth.TakeDamage(MaxHealth)`.
6.  **Death Flow:**
    `PlayerHealth` (OnDeath) → `GameManager` (устанавливает `GameState.GameOver`, `Time.timeScale = 0`).
    `GameManager` (в ответ) → отключает `Input`, останавливает `WorldMover`, `ObstacleSpawner` и `BonusSpawner`.
